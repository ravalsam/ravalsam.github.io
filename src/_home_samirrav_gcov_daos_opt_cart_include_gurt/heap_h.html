58  * Binary heap
59  *
60  * The binary heap is a scalable data structure created using a binary tree. It
61  * is capable of maintaining large sets of objects sorted usually by one or
62  * more object properties. User is required to register a comparison callback
63  * to determine the relevant ordering of any two objects belong to the set.
64  *
65  * There is no traverse operation, rather the intention is for the object of the
66  * lowest priority which will always be at the root of the tree (as this is an
67  * implementation of a min-heap) to be removed by users for consumption.
68  *
69  * Users of the heap should embed a d_binheap_node_t object instance on every
70  * object of the set that they wish the binary heap instance to handle, and
71  * required to provide a d_binheap_ops::hop_compare() implementation which
72  * is used by the heap as the binary predicate during its internal sorting.
73  *
74  * The implementation provides an optional internal lock supporting, user can
75  * select to use its own external lock mechanism as well.
76  */
