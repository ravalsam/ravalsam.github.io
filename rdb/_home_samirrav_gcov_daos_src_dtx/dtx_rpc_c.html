544 /**
545  * Commit the given DTX array globally.
546  *
547  * For each DTX in the given array, classify its shards. It is quite possible
548  * that the shards for different DTXs reside on the same server (rank + tag),
549  * then they can be sent to remote server via single DTX_COMMIT RPC and then
550  * be committed by remote server via signle PMDK transaction.
551  *
552  * After the DTX classification, send DTX_COMMIT RPC to related servers, and
553  * then call DTX commit locally. For a DTX, it is possible that some replicas
554  * have committed successfully, but others failed. That is no matter. As long
555  * as one replica has committed, then the DTX logic can re-sync those failed
556  * replicas when dtx_resync() is triggered next time
557  */
